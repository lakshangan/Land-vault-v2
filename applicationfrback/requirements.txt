1. Project Overview
Land Vault is a production-grade, full-stack Web3 protocol for tokenizing Real World Assets (RWA) across five asset categories: Land, Renewable Energy, Infrastructure, Timber/Forestry, and Real Estate. The platform enables fractional ownership, on-chain yield distribution, DeFi lending/borrowing, an asset-backed stablecoin, liquidity pools, yield farming, and DAO governance — all deployed on BNB Chain.
1.1 Core Platform Capabilities
Asset Tokenization
Convert any RWA (land, solar farm, telecom tower, forest, REIT) into ERC-721 NFTs + ERC-20 fractional ownership tokens.
Fractional Investment
Global investors buy fractional tokens from as little as $10, gaining proportional yield rights.
DeFi Yield Engine
Asset-generated income (rent, energy revenue, toll fees, timber sales) is auto-distributed to token holders via ERC-4626 vaults.
Lending & Borrowing
Token holders collateralize their RWA tokens to borrow stablecoins, with protocol-managed liquidation.
AMM Liquidity Pools
Native DEX with asset-specific liquidity pools enabling token trading without centralized exchanges.
Land-Backed Stablecoin
LVT — a stablecoin minted against RWA collateral, over-collateralized for price stability.
DAO Governance
Token holders vote on protocol parameters, new asset onboarding, fee structures, and geographic expansion.
AI Valuation Engine
On-chain oracle + off-chain ML model providing real-time asset valuations, yield forecasts, and risk scores.
Document Registry
Legal documents stored on IPFS/Arweave with on-chain content-hash verification.

7.2 Contract Interactions Flow
Asset Submission
Owner calls AssetRegistry.submitAsset() → stored off-chain → Validator calls AssetRegistry.mintAsset() → NFT minted, AssetToken deployed, AssetVault deployed
Buy Fractional Token
Investor calls AssetToken.buy(amount) → ERC-20 transferred, USDT sent to YieldDistributor → Holding recorded on-chain
Yield Distribution
Asset operator calls YieldDistributor.depositYield(assetId, amount) → snapshot taken → token holders call claim() → USDT transferred to wallet
ERC-4626 Vault
Investor calls AssetVault.deposit(assets) → shares minted → yield auto-compounded in vault → withdraw() redeems shares for assets + yield
Lending/Borrow
Borrower calls LendingPool.depositCollateral(rwaToken, amount) → calls borrow(stable, amount) → health factor computed → loan active
LVT Mint
User calls CollateralVault.lockCollateral(rwaToken, amount) → calls LVTStablecoin.mint(amount) within collateral ratio → LVT sent to wallet
Liquidity Add
LP calls AMMPool.addLiquidity(tokenA, tokenB, amountA, amountB) → LP tokens minted → position tracked
Governance Vote
Holder calls GovernanceDAO.castVote(proposalId, support) → vote weight = LVT balance + RWA token balance → quorum check on deadline

8. Authentication, KYC & Security
8.1 Wallet-Based Authentication (Sign-In With Ethereum)
Step 1: Frontend requests a nonce from POST /api/v1/auth/nonce (nonce stored in Redis with 10-min TTL)
Step 2: User signs the nonce with their wallet (EIP-191 personal_sign)
Step 3: Backend verifies signature via ethers.verifyMessage(), issues JWT access token (15min) + refresh token (7d)
Step 4: Access token stored in memory (never localStorage). Refresh token in httpOnly cookie.
All authenticated API calls send Authorization: Bearer <token> header
Chain ID verification: only BNB Chain (56) and BSC Testnet (97) accepted
8.2 KYC/AML Requirements
All users must pass KYC before investing (investing CTA is gated)
KYC provider: Sumsub or Persona (third-party integration via API + webhook)
KYC levels: Basic (identity verification) for investments up to $10,000; Enhanced (proof of address + source of funds) for above $10,000
KYC status stored in DB. Updated via webhook from KYC provider.
Restricted countries: OFAC sanctions list enforced at KYC + IP level
8.3 Security Requirements
All API endpoints: CORS restricted to allowed origins only
Rate limiting: 100 req/min per IP (public), 300 req/min per authenticated user
SQL injection: prevented by Prisma ORM parameterized queries
XSS: Content-Security-Policy headers on all responses
Smart contracts: OpenZeppelin ReentrancyGuard on all value-transfer functions
Smart contracts: Ownable + AccessControl roles (VALIDATOR_ROLE, ADMIN_ROLE, PAUSER_ROLE)
Smart contracts: Pausable mixin for emergency stop
Multi-sig: ProtocolTreasury + contract upgrades require 3-of-5 multisig (Gnosis Safe)
Private keys: NEVER stored in code. Loaded from environment secrets only.
Audits: contracts must pass internal review before mainnet. Third-party audit recommended before public launch.
